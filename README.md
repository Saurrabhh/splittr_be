# Splittr Backend (`splittr_be`)

A Go-based backend service for **Splittr**, a bill-splitting application. This service features clean architecture, type-safe SQL interactions using `sqlc`, schema migrations with `goose`, live reloading via `air`, and token-based authentication integrated with the Firebase Admin SDK.

---

## 🚀 Tech Stack

- **Language:** Go (1.26.3)
- **Database:** PostgreSQL (16)
- **Routing:** [go-chi/chi](https://github.com/go-chi/chi) (v5)
- **SQL Generator:** [sqlc](https://sqlc.dev/) (v2)
- **Migrations:** [goose](https://github.com/pressly/goose)
- **Authentication:** Firebase Admin SDK
- **Hot Reload:** [air](https://github.com/air-verse/air)
- **API Documentation:** [swag](https://github.com/swaggo/swag) (Swagger UI)

---


Before getting started, make sure you have the following installed on your machine:
- **Go** (v1.26.3 or later)

- **swag** CLI: `go install github.com/swaggo/swag/cmd/swag@latest`

## 🛠️ Configuration & Setup

Follow these steps to configure and spin up the project locally:

### 1. Configure Firebase Admin SDK
1. Go to your **Firebase Console** -> **Project Settings** -> **Service Accounts**.
2. Generate a new private key and download the JSON file.
3. Save the downloaded file to the root of the project with the filename `firebase-key.json` (this file is already included in `.gitignore` to prevent leaking credentials).

### 2. Spin up PostgreSQL
Use Docker Compose to run a local PostgreSQL instance in the background:
```bash
docker compose up -d
```

### 3. Run Schema Migrations
Apply database migrations using `goose` against your local PostgreSQL database. The repository provides command shortcuts in the `Makefile` to automatically load env files:

```bash
# Run migrations for the local environment
make goose-up local
```

*(You can run other goose commands similarly: `make goose-status local` or `make goose-down local`.)*

### 4. Generate Database Code (Optional)
If you add or update SQL queries inside `internal/db/queries/` or migration schemas in `internal/db/migrations/`, generate the Go boilerplate using `sqlc`:
```bash
sqlc generate
```

## 🏃 Running the Application

### Development (with Hot Reload)
Run the application using the `Makefile` shortcut. This loads variables from `env/local/.env` and triggers `air` to watch for file changes, rebuild the binary on-the-fly, and restart the API server:
```bash
make local
```

### Production Build & Run
To compile a production binary and execute it:
```bash
make prod
```

## 🔌 API Playground & Endpoints (Swagger)

We use **Swagger UI** for API documentation. The interactive playground displays all request/response JSON schemas, authentication definitions, and lets you test endpoints directly from your browser.

### Accessing the API Playground
Start your local server (`make local`) and navigate to:
```text
http://localhost:8080/swagger/index.html
```

### Regenerating Documentation
If you update request/response structs or handler functions, regenerate the Swagger schema files by running:
```bash
make swagger
```

### Authentication
All v1 endpoints (except `/health`) require authenticating with a **Firebase ID Token** sent via the `Authorization` header as a Bearer token:
```text
Authorization: Bearer <FIREBASE_ID_TOKEN>
```

### Core API Groups
- 👤 **Users & Friends (`/v1/users`, `/v1/friends`)**: Register profile, fetch current user info, search & add friends, remove friendships.
- 👥 **Groups (`/v1/groups`)**: Create & join bill splitting groups, manage members, update member roles.
- 💸 **Expenses & Balances (`/v1/expenses`, `/v1/balances`)**: Create splits (equal/exact/percentage), settle debts, fetch global or group-simplified balances.
- 📊 **Activities (`/v1/activities`)**: Browse group activity logs and audit trails.
- 🔔 **Notifications (`/v1/notifications`)**: Retrieve personal notifications, mark items as read.
